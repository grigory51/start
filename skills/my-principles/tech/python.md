# Python

> Принципы использования Python. Общие инженерные принципы — в [SKILL.md](../SKILL.md), здесь только специфика языка и стека.

## Инструменты и версии

- **Python 3.12+.** Backend, инфра, CLI.
- **Пакет-менеджер: `uv`.** От `requirements.txt` уходить в пользу `uv` («от requirements.txt избавиться надо!»).
- **CLI: `click`.**
- **Async: `asyncio` / FastAPI** для сервисов.
- **Модели/сериализация: Pydantic v2, marshmallow (`Schema`).**
- **Типы: mypy** — код должен быть mypy-чистым.

## Принципы

- **Типизация строго и явно.** Аннотации обязательны. Вместо строковых констант — `Literal[...]` / enum-подобные типы (`role: Literal["edge-router", ...]`, а не `EDGE = "edge-router"`).
- **(Де)сериализация — явно в модели.** Преобразования держи в модели/датаклассе (Pydantic, `Schema`), а не прячь в разрозненные хелперы. Преобразование в датакласс/модель делай на этапе создания dict.
- **Ошибки наружу — полный traceback,** а не один `msg` («приходит ошибка, но только msg, хотелось бы видеть весь traceback»).
- **Тесты для логики обязательны,** по паттернам проекта. Но тесты — следствие кода, не источник истины для проектирования.

## Анти-паттерны

- `requirements.txt` вместо `uv`.
- Магические строки вместо типизированных `Literal`/enum.
- Сериализация, размазанная по хелперам вместо модели.
- Проглатывание исключений; ошибка наружу без traceback.
- Локальные импорты, которых можно избежать; циклические зависимости.
