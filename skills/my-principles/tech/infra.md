# Инфра и dev-окружение

> Принципы инфраструктуры, локальной разработки и деплоя. Общие принципы — в [SKILL.md](../SKILL.md).

## Инструменты

- **Контейнеры: Docker / docker-compose.**
- **Локальная разработка: Tilt.**
- **Деплой: serverless (Yandex Cloud).** Важна быстрая загрузка контейнера — оптимизируй cold start.
- **Монорепо: Yandex Arc.** *(контекст конкретного проекта.)*

## ⛔ Удалённые серверы — только с явного разрешения

**Никогда без явного разрешения не делай ничего с удалёнными серверами.** Запрещено
по умолчанию, без отдельного «да» на конкретное действие:

- выполнять команды по SSH (включая «безобидные» вроде `ls`, `cat`, `systemctl status`);
- запускать Ansible / playbook'и, Terraform apply, любые оркестраторы инфры;
- деплоить, перезапускать сервисы, править конфиги, миграции, крон;
- всё, что **может повлиять** на удалённое состояние — даже потенциально, даже «только посмотреть».

Если задача требует касания удалёнки — **сначала спроси и опиши, что именно собираешься
запустить** (точные команды/хосты), и дождись явного подтверждения. Локальные dry-run,
генерация скриптов/плейбуков «на проверку» без запуска — можно.

## Принципы

- **Без костылей.** Есть рекомендованный способ — переезжать на него («чтобы без костылей»).
- **Быстрый cold start** для serverless: лёгкие образы, минимум на старте.
- **Cache invalidation по версии.** Не держать устаревший кеш после обновления.

## Анти-паттерны

- Костыли и обходные пути там, где есть рекомендованный способ.
- Дублирование конфигов между окружениями.
- Тяжёлые образы, медленный cold start на serverless.
