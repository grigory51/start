# Frontend (TypeScript, React, 3D, UI/UX)

> Принципы фронтенда, языка и визуального качества. Общие принципы — в [SKILL.md](../SKILL.md).

## Инструменты и стек

- **Язык: TypeScript** (и Node-тулинг, где уместно).
- **React + Vite** (или Next.js).
- **3D: Three.js / Babylon.js.**
- **Редактор конфигов в UI: MonacoEditor.**
- **UI-кит: gravity.** *(контекст конкретного проекта — не универсальный закон.)*

## TypeScript

- **Типизация строго и явно.** Никаких `any` без причины. Вместо строковых литералов вразнобой — union-типы / `as const`, по аналогии с `Literal[...]` в Python.
- **Не дублировать валидацию фронт↔бэк.** Если бэк уже валидирует — не повторять на фронте. Источник истины один.
- **(Де)сериализация — явно.** Типы данных с бэка описывать явно, преобразования держать в одном месте, не размазывать.

## UI/UX — владелец придирчивый визуальный критик

Проверяй как он:

- **Выравнивание и отступы — по пикселю.** «Плохое выравнивание по вертикали», «проёбаны отступы, кнопки вплотную» — блокеры, не придирки.
- **Никаких лишних скроллов.** Контент помещается в окно; скролл — только где осмыслен. «3d-сцена не больше окна, скролл только где параметры».
- **Тёмная тема обязательна** — инпуты, 3D-сцены, все компоненты. Не опционально.
- **Прямая манипуляция.** Где уместно — интерактив на самом объекте/сцене, а не только в боковом UI. «Хочу тыкать и двигать прям на модели».
- **Согласованность с макетами и прошлым поведением** — источник истины. «Раньше был справа и корректно написан» → там и остаётся.
- **Состояния без гонок.** Не показывать «Choose Device» одновременно с лоадером; не фуллскрин-лоадер на модалку; loading/empty/error/hover/focus — все продуманы.
- **Прогресс внятный.** Прогресс-бар на всю ширину, внутри — проценты и текущая команда.
- **Без магии в поведении.** Перпендикулярные стены не двигаются, параллельные нельзя переставлять — UI моделирует реальные ограничения.
- **Мобилка.** Фикс-хедеры, дерево, меню — проверь, что не разъезжается на узком экране.
- **Reference-driven.** Ориентируйся на качественные референсы (`bento3d.design` и т.п. — пример конкретного проекта), избегай generic-шаблонности.

## Анти-паттерны

- Generic AI-эстетика: шаблонные градиенты, безликие лейауты, кривые отступы, лишние скроллы.
- Фуллскрин-лоадеры, лишние ремаунты, UI-гонки состояний.
- `any` как способ заглушить типы.
- Дублирование констант/проверок/валидации между фронтом и бэком.
- Циклические зависимости между модулями.
