# Глобальные инструкции (все проекты)

Этот файл линкуется в `~/.claude/CLAUDE.md` менеджером `claude-agents` (`make up`).
Правится здесь, в репозитории — не в `~/.claude` напрямую.

## Язык

Всегда отвечай пользователю по-русски. Код, имена, команды, технические термины — как есть.

**Субагенты.** При запуске любого субагента (Task/Agent, Plan, claude-code-guide и пр.) явно
добавляй в его промпт требование отвечать по-русски — субагенты НЕ наследуют это правило
автоматически. Если субагент вернул ответ на английском, переведи его пользователю на русский,
не пересказывай по-английски.

## Принципы

Всегда следуй навыку `my-principles` (личные инженерные принципы, стиль, инфра) —
он линкуется в `~/.claude/skills/my-principles/`. При проектировании, написании, ревью и
документировании кода применяй его, даже если skill не подтянулся автоматически.

## Область поиска

Когда пользователь просит что-то найти (код, файл, упоминание, символ), по умолчанию ищи
**в рабочей директории проекта**. За её пределы (другие репозитории, ~, вся система) выходи
только если пользователь явно об этом попросил.

## Комментарии в коде

Комментарии пиши формально, нейтральным техническим языком — без жаргона, сленга, мемов и
фамильярности. Объясняй «почему», а не «что». Подробнее — `my-principles` → «Стиль кода».

## Удалённые серверы

Никогда без явного разрешения не делай ничего с удалёнными серверами: ни SSH-команд (даже
read-only), ни Ansible/Terraform, ни деплоя/рестарта/правки конфигов — ничего, что может
повлиять на удалённое состояние. Сначала спроси с точными командами/хостами, дождись «да».
Подробно — в `my-principles` → `tech/infra.md`.
